Week 2 - Authentication, authorization , access control

What is Identification?

This is when you say who you are, e.g, username.

What is Authentication?

This is when you confirm/prove who you are, e.g, password

What is Authorization?

Authorisation is the function of specifying access rights/privileges to resources related to information security and computer security. They should use least privileges which is when you get the minimum access to the resources that you need to do your job.

Types of authentication?

  • User knowledge - pin, password
  • A token - Keycard
  • Static biometrics - Finger, retinal, face
  • Dynamic biometric - Voice, hardwriting

What is password salt?

Salting is a cryptographic way to store passwords in a strong way, they are a set of random characters that are added to the end of a password before they are hashed. They are added automatically after a user types in the password. Salts are stored in plain text with the hash output so the website knows what salt to use.

What is access control?

To control who can access which resource, and to stop unauthorized access. So only authorized access can do the authorized task.

What is MAC (Mandatory access control)?

A model of access control that is controlled by the admin. When someone makes a asset eg folder, there is not owner.

This model is extremely secure as only one person can decide who has access to what. But is more complex and is hard to scale because you have to keep changing which subject have access to the object.

What is DAC (Discretionary access control)?

A model of access control. Every object has an owner, the owner has full control of who can access or modify the object. The admin can access the file even if the owner didn't give permission.

This is the easiest to use and to implement. The issue is it doesn't scale well because it will have so many different owners and so many different files to give access to.

What is RBAC (Role based access control)?

A model of access control, users are put in a group and each group is only allowed specific assets.

This is simple and scales easy but you have to trust every person in the group.

What is ABAC (Attribute based access control)?

Each user has a number of attributes that can either allow them or deny them from accessing a object.

Ths gives you more granular control over who can access the file but is more complex as you have to set up a number of attributes for each person.

Explain the relationship between Authentication, Authorisation and an Access Control Policy?

Ensures security and access control policies are in place to protect confidential information e.g. customer data.

The relationship between all three is that it keeps the system secure by having access control policies in place to keep the systems confidentiality, integrity and availability .

Week 3 - Encryption

What is cryptography?

When you encode information using an algorithm.

What is encryption?

The process of changing normal text to cipher text

What is decryption?

The process of changing cipher text to normal text.

What is cipher text?

Text that is not readable by humans.

What is a hash?

A mathematical function that changes a text into a random output string that has fixed length. The same input string will always produce the same output string, if the input string is changed by a single charter the output string will be completely different.

Hash is a one way encryption, you cant decrypt it.

How can you decode a hash?

Because hashes always produce the same output string, by comparing to the database of the common passwords, you can reverse engineer what the password was. This is called a rainbow table.

What is cipher?

The algorithm used to encrypt and decrypt text.

What is key?

The parameters to guide the cipher

Cipher and key

You need both the cipher and the key to decrypt data

Substitution cipher

When you change a letter for the alphabet to another letter. Caesar is a type of Substitution cipher

This is when shift the letters in an alphabet a certain amount to encrypt the message. Eg Lets shift minas 3 letters this would give me plqdv. To decrypt it you apply the sift.

This can be cracked because each language has a frequency for each letter for eg in the uk the letter e is the most common letter used. To break it you count the frequency of each letter and check how far the shift has shifted. This is called frequency analysis.

It can also be cracked by brut force as their are only 25 keys.

What is transposition cipher

This is a pattern of how the order of characters using rows and columns to change text eg when you write the message in row and encrypt the message in columns.

This is still suspectable to frequency analysis and brute force.

What is cryptoanalysis

When you decrypt data without knowing the cipher or the key used

Types of encryption

Symmetric encryption - only private key. You use the same key to encrypt and decrypt. This processes text into fixed block sizes. E.g, DES, 3DES, AES. The issue is you need to securely swap the keys. This is faster than asymmetric encryption, and their isnt a size limit of the message.

Asymmetric encryption - A public and a private key. The public key can be distributed freely. The key used to encrypt is different to the key used to decrypt. The person encrypts an outgoing message using the receivers public key, but only the receivers private key can decrypt it. This can be used for digital certification and SSL. Its slow and has a message size limit, but it is safer.

DES (data encryption standard)

Divides messages into blocks of 64 bits, and uses a 56 bit key. This was crack by brute force.

3DES (3data encryption standard)

Repeats Des 3 times so the key is 168 bits. This is very secure and brute force would take a long time, the issue with it is that it is slow.

AES (Advanced encryption standard)

This has a block length of 128 bits and the key is 128,192, or 256. Very secure and faster than 3DES.

What is SSL

Secure Socket Layer this uses public key to secure channel over the internet.

What is S-HTTP

Secure hypertext transfer protocol is an extended version of HTTP, that uses ssl to provide encryption of messages between server and user. This works by providing an encrypted layer between the website and the browser making sure hackers cant access the data.

Week 3 - Secure coding

What are the main causes of vulnerabilities and how can you protect against them?

Bugs and logic flaws are the main causes of vulnerabilities. To protect against them you need to have different testing and development teams, a fresh pair of eyes can help spot new issues. Assessment process for unknown vulnerabilities. Automated and manual tools should be used. The use of coding standards and practices.

What is architectural analysis?

The functionality and structure of the system, for example the interaction between the program and the users, what operation are users allowed to do/ what can they assess.

What is buffer overflow in C/C++?

When a user writes a variable into a piece of memory that has a certain length. But the variable is bigger than the specified memory so it overwrites other data.

What is int overflow in java?

A int in java is 32 bits so, when a user attempters to go over 32 bit it he can break the program.

What is sql injection?

This lets an attacker have access to the database using sql through an input. This is usually data they are not allowed to view, the attacker might be also abel to modify or deleted the data, causing permanent damage.

What is automatic code review?

A software that can find vulnerabilities in a program. This is faster than manual code review and can be optimized according to a standard. However, it might produce false positives and false negatives, and might be expensive to use.

What is manual code review?

This is when someone deep dives into the code to see if there are any vulnerabilities. They check for logic errors and things automatic code review tools might not be able to find. There are tools that help with manual code review for example to make reading the code easier or to be able to comment. This requires extra time as it isn't done automatically.

Week 4 - Privacy engineering

What is Privacy engineering?

This is a software engineering related discipline on how you handel users data, it must follow regulations and must be privacy by design.

Why is Privacy not security?

A lose of privacy doesn't mean a data breach, and having good security doesn't guarantee privacy. However it is important to have good security to archive privacy.

Who can undertake privacy protection

  • Government laws
  • Business code of conduct
  • By users through PET(Privacy enhanced tech)

What is privacy impact assessment(PIA)?

This is done by an independent entity that is not linked to the project. To see how new data collection will affect the users. This is done to consider privacy issues while making the project.

What is Privacy by design (PbD)

These are 7 principles that says how we should deal with data

  1. Proactive not reactive - stop data leaks before it happens
  2. Privacy is the default setting
  3. Privacy embedded in design - while building the software think of privacy
  4. Fully functional
  5. Security - data is controlled and stored securely
  6. Visibility & Transparency
  7. Respect for user privacy

What are PET - Privacy enhanced technologies

Software users use to protect their data

What is k-anonymity?

When you hide peoples PII (personal identifiable info) but revealing characteristics of the data, to for example researchers.

What is a linking attack?

A linking attack is when someone has been k-anonymised and is in a dataset. A bad actor ties to link the data set to a person. To prevent this you Increase the degree of generalization when releasing a dataset.

Week 6 - Network security

What is Security intrusion

When you enter into a network, unauthorized

What is intrusion detection system (IDS)

This detects and analyses info from a network to see for a possible intrusion. This can be hardware or software and is placed in multiple areas of the network.

What are the three types of IDS

  1. Host-based (HIDS)- This monitors a single host like a computer, for any suspicious activity.
  2. Network based IDS (NIDS) - This monitors specified parts of the network for and suspicious activity. They should be put in strategic points of the network.
  3. Distributed or hybrid IDS - This is on both the host and the network there is a central analyser that detects suspicious activity

What are the three logical components of IDS

  1. Sensors - collect data
  2. Analyzer - finds out if there was an intrusion that happened.
  3. User interface - log of things that happened / alerts system admin when a breach happens

How can you generate a log summary report

  • Automated analysis
  • Manual log analysis

IDS requirements

  • Ran continually
  • Minimal overhead on system
  • Can scale to monitor a large number of system
  • Be fault tolerant
  • Can be configured to system polices.
  • Flexibility

What are the two analysis approaches to IDS

  • Anomaly detection - This collects data about behavior of real users over period of time and compare it to current behavior to check if it is an intruder.
  • Signature/heuristic detection - Uses data of malicious activity and compares with current behavior , can only identify known attacks.

What data do IDS collect?

  • System call traces
  • Log file
  • File integrity checksum
  • Registry access

NIDS characteristics

  • Monitors traffic at selected points on a network
  • Examines traffic packets

What is a honeypot?

This is a fake system to lure potential cyber attackers away from critical software, this collects attackers activity and tries to get them to spend long enough so the admin can respond. The system is filled with fake info and the resources have no production value.

What are the two types of honeypots?

  1. Low impact honeypot - It emulates a system well enough to fool someone initially but it isn't that realistic as its not a full system.
  2. High impact honeypot - A real system with a full OS, services and apps. Its a more realistic target that may take an attacker longer to work out that it is a fake system. However it requires more resources and if compromised it could be used to attack other systems.

What is a firewall?

A system that is used to prevent unauthorized access from entering into a network. This is done by filtering private info coming in the internet. The firewall blocks unwanted traffic and lets wanted traffic.

A firewall works by filtering the incoming traffic and determine by the rules (access control list/ Access polices) given if it is allowed to enter or not, this is controlled by system admin.

The firewall rules can be based on

  • IP address
  • Domain names
  • Protocols
  • Ports
  • Key words

There are both host based firewall and network based firewalls

Firewall capabilities

  • Defines a single choke point
  • Provides a location for monitoring

Firewall limits

Can't protect against attacks bypassing the firewall

Cant fully protect against internal threats

What is an IPS (intrusion prevention system)?

The tools that blocks intrusion, this is a control system that reads packets and compares them to common threats. The IPS can do everything the IDS does but it can also block an attack, so this activity defends the network. So it detects ands sends an alert if there is an attack. Its the next evolution of IDS.

Good security

To have good security you should never rely on a single countermeasure, you dont wabt one point of failure

Minimal attack surface so you should have any unnecessary features or services.

Monitors network behavior.

Week 6 - Wireless security

What is 802.11?

This is the standard for WLAN the most common brand name is WIFI

Why is wireless more vulnerable than wired?

You don't have to be close to the network to hack it. With a wired network you need physical access to get in the network, you don't need that with wireless.

Wireless Network components

  • Wireless client device - Phone, laptop
  • Access point - Router
  • transmission medium - radio weaves for data transfer (the source of vulnerability)

How can you deal with eavesdropping in a wireless network ?

Use single hiding techniques like, Turn off SSID broadcasting, Reduce signal strength, Locate wireless access points in the interior of the building, away from windows and exterior walls.

Encryption is effective against eavesdropping to the extent that the encryption keys are secured.

Use antivirus, antispyware software and a firewall.

Change your router’s pre-set password for administration.

Allow only specific computers to access your wireless network.

Mobile device security threats

  • Device might be stolen
  • Use of untrusted mobile devices
  • Use of untrusted networks
  • Use of applications created by unknown parties

Week 7 - Malware

What is Malware?

This is software that was created to have malicious intent, and usually performs unwanted activity. This can affect , applications, Utility programs, Kernel-level programs (eg operating systems)

What two ways can you classify a malware?

  1. How it spreads to reach the desired target
  2. The actions it takes once it reaches the target

You can't fully classify, as they are not mutually exclusive so malware as it uses multiple techniques

How can the malware spread/Propagate?

They could have an executable file that contents a virus in it.

They could exploit a software vulnerabilities to add malware. This could be done on the host or on the network, a worm could then replicate

ring attacks that convince users to bypass security mechanisms to install trojans or to respond to phishing attacks.

What payload actions do malware perform

  • Corrupt of system or data file
  • Theft of service to make a botnet
  • Theft of system/files, password key logging or spyware
  • Stealth malware tries to hide from the ant virus

What is a blended attack?

When you use multiple of the methods above.

What are attack kits/crimeware?

These are tools that spread and payload a malware for you, it can be easily customized with the latest discover vulnerabilities to exploit the window of opportunity.

What is a virus

Malware and virus aren't the same thing, however virus is a type of malware.

A virus is a program that infects other programs by changing them maliciously.

Virus distinguishing features include:

  • Reproduce its own code
  • Attaches itself to other files (cant survive by itself)
  • Spreads when the executable file is executed
  • They thrive on lack of user authentication and access controls thats why they were so dominant in the early days.

Virus structure include:

  1. Dormant phase - when the virus is ideal not all the viruses have this stage.
  2. Infection mechanisms - How the virus spreads or propagate.
  3. Trigger/Logic bomb - The event that causes it to activate the payload
  4. Payload - What does the virus do besides spread.

What are Macro and scripting virus

These are usually exploited in microsoft office or a official app that lets you run mini code. The attacker runs malware in that mini code, this can be done through a word document or excel that usually spread through a phishing email and gets you to click enable macros, by enabling the macros you allow the documents to run malware. Macros are stored as visual basic code or XLM so it is easily for a hacker to put malicious code on it, to do things like execute a power shell command or download a malware. XLM its harder for antivirus to detect because the technology is so old. Because macro viruses infect user documents rather than system programs, traditional file system access controls are of limited use in preventing their spread.

What are worms

A program that tries to infect more machines. To replicate itself a worm needs access to systems like email, file sharing, reverse shell or team viewer. It has the same phase as the virus. The propagation phase tries to find access to other systems to infect.

After a worm gets in a system, it goes to the network and spread once it finds other systems it infects them. Everytime a system connects to a new network the process repeats.

Bots and botnets

A bot is a machine that is controlled by an attacker remotely. They dont have to affect the system it takes some system resources to mine crypto or for a DDOS attack. It is usually a larger network of bots and their is a master bot that controls what the system does.

The attacker controls the Command and Control server (C&C) server, all bots contact the C&C and report their status and receivers commands.

The issue with the C&C is the it provides a single point of failure. To prevent that most attackers have multiple C&C servers. Large botnets usually are broken up into smaller once so each bot only knows one C&C server.

Their are two approaches to C&C which are:

  • Push style - botmaster sends command to the bot.
  • Pull style - bot asks master for commands

Address for each C&C server must be available to each bot this includes binary, config and must be frequently updated.

What is DOS attack?

Denial of service this is when a devices floods the network/website with traffic, making it slow or unaccessible.

This dosnt work as much any more because servers got good at handling a lot of traffic.

What is a DDOS attack

Distributed danial of service. This is the next upgrade to DOS it uses multiple devices (usually botnets) to flood the network making it unavailable.

What is a trojan horse

This is when a malicious program is hidden in an application, it preforms a wanted function but also does malicious activity.

This requires you to run the program but it can't self replicate.

What is spyware?

Malware that secretly spys on users and collects data without their concent.

This is hidden from users and is hard to detect. Leaks info like passwords and card info.

What are key loggers

Malware that records users key strokes. This runs in the background. Wants to capture data like password. They are easy to write and difficult to recognize by antivirus.

What is Drive by download

This is a malware installs automatically do you device without your consent, this usually happens when you when you download an software and multiple other software get downloaded.

Week 8 - Cloud computing

What is cloud computing?

A system (SAAS),platfom (PAAS) or infrastructure (IAAS) that is on the internet.

Public cloud

infrastructure is made available to the general public and is owned by an organization selling cloud services. The cloud provider is responsible for both the cloud infrastructure and the control of data. Its cheap but might have security concerns.

Private cloud

This is implemented within the internal IT environment of the organization. The organization may choose to manage the cloud in house or contract the management function to a third party. The cloud servers and storage devices may exist on premise or off premise. It is more secure but more time consuming and more expensive

Community cloud

Is a commination of public and private cloud. It has the restrictions like the private cloud but has shared resources like the public cloud. The cloud infrastructure is managed by the organization.

What is hybrid cloud?

This is made up of two or more clouds that are septate bound together by a software that allows you to transfer data. So the sensitive info can be in a private cloud and the less sensitive can be in the public cloud, this is good for small bushiness as it is cheaper but also secure.

Data loss or leakage - Have a strong access control, Encrypt and protect integrity of data, Implement strong key generation, storage and management, and destruction practices

Week 9 - OS

OS security

It is possible for a system to be compromised during installation process before the patches are installed, so building and deploying a system should be planned and designed to counter this threat.

The process must

  • Assess risks and plan the system deployment
  • Secure the OS and key apps
  • Make sure all important content is secure
  • Have appropriate network protection

What strategies can you implement to keep the OS secure?

  • White list approved apps
  • Patch third party apps and OS vulnerabilities
  • restrict admin privileges
  • Have access control
  • How can users authenticate

What is OS hardening?

This is when you make your system more secure by having the latest patches, by having all the apps up-to-date. After you do this you have to regularly monitor to make sure that you don't have any vulnerabilities.

Other ways you can harden your system is by removing unnecessary services, applications. Adding access control to it. You could configuring resource controls.

Add things like IPS or IDS antivirus to the system.

Process of installing a new system

  1. Installing the OS on a protected network
  2. Harden the OS
  3. Install the minimum apps/ software
  4. Secure boot
  5. System must be kept up to date
  6. Validate all patches and test security

Why is removing unnecessary software good?

Fewer software the less risk you have to run in a vulnerability

Security maintenance

This is the process of maintaining security continually

  • Monitoring and analyzing log info
  • Performing backups
  • regularly testing system security

What is logging?

Tells you about what happened to your system and keeps records of it. In an event of an attack you can identify what happened. Automated analysis is preferred. Takes a lot of storage

Virtualization?

Software that lets you run a virtual os inside an os, you can have multiple os in one system. This raises security concerns because they might be able to access the host system.

What is a hypervisor

The middle man between the hardware and the VM, this helps allocate resources.

Vm Security issues?

Guest OS isolation - make sure the programs in the VM only access resources allocated to it.

What is Security auditing?

An independent review of systems records and activities. To see if your system has good system controls, security polices, and if any breaches happened. They also might recommend changes to the systems.

This is done to have accountability. system entities that initiate or participate in security-relevant events and actions

What is a security audit trail

A record of system activities that can enables you to see what happened. This can be used to see what happened leading event involving security breach. This can be sen from the beginning to the end of the event.
